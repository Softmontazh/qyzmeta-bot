## 🔧 ТЕХНИЧЕСКИЕ ДЕТАЛИ

### BUS_ID Формат
```
BUS_XXXXXXXXXXXXXXXX
├── BUS_ - префикс (4 символа)
└── XXXXXXXXXXXXXXXX - 16-символьный хеш (MD5 от file_id)
```

### Workflow Медиафайлов
1. **Пользователь** загружает фото/видео в заявку
2. **FSM** сохраняет медиа в BUS группу через `bus_service.save_image()`/`save_video()`
3. **База данных** сохраняет оригинальный `media_id` и сгенерированный `bus_media_id`
4. **Сервисник** получает уведомление с кнопкой "Подробнее"
5. **Просмотр медиа** через BUS систему доступен всем ботам в группе

### Безопасность
- BUS группа (-1002836867857) служит централизованным хранилищем
- Доступ к медиа только через валидные BUS_ID
- Автоматическая генерация уникальных идентификаторов
