## 🔧 Технические особенности

### BUS ID Format
```
BUS_XXXXXXXXXXXXXXXX (20 символов)
- Префикс: BUS_ (4 символа)  
- Hash: 16 hex символов (A-F, 0-9)
```

### Workflow медиафайлов
1. **Пользователь загружает медиа** → Сохраняется оригинальный `file_id`
2. **Бот отправляет в BUS группу** → Получает новый `file_id` из группы  
3. **Генерируется BUS_ID** → Сохраняется в базе данных
4. **Сервисник получает уведомление** → Может просмотреть медиа через BUS

### Структура базы данных
```sql
-- Поле уже создается автоматически при создании таблицы
CREATE TABLE offers (
    ...
    bus_media_id VARCHAR(20) NULL,
    ...
);
CREATE INDEX ix_offers_bus_media_id ON offers(bus_media_id);
```
