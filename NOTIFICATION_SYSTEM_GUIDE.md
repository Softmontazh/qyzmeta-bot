# 🔔 Система уведомлений заявок

## 📋 Обзор системы

Система автоматических уведомлений для управления жизненным циклом заявок в Qyzmeta-Bot.

## 🔄 Жизненный цикл заявки

### Статусы заявок:

1. **🔔 ACTIVE** (Активная) - заявка создана, ожидает обработки
2. **⏳ IN_PROGRESS** (В работе) - заявка принята поставщиком услуг
3. **✅ COMPLETED** (Выполнена) - заявка успешно выполнена
4. **❌ CANCELLED** (Отменена) - заявка отменена
5. **📦 ARCHIVED** (Архивная) - заявка помещена в архив

## 👥 Участники процесса

### 🏠 Жители (заявители):
- Создают заявки через команду `/add_offer` или "Создать заявку"
- Получают автоматические уведомления о смене статуса заявки
- Могут архивировать свои заявки

### 🔧 Поставщики услуг (сервисники):
- Получают уведомления о новых заявках в их категориях
- Могут изменять статус заявок через команду `/manage_offer_status`
- Имеют доступ только к заявкам в своих ЖК

### 👑 Администраторы ЖК:
- Могут добавлять и управлять поставщиками услуг
- Имеют доступ к управлению заявками в своих ЖК

## 📱 Команды и функции

### Для жителей:
- **Создать заявку**: `/add_offer` или "Создать заявку"
- **Мои заявки**: через меню бота
- **Архивировать заявку**: через список заявок

### Для поставщиков услуг:
- **Управление статусом**: `/manage_offer_status`
  - Введите номер заявки (например: `123`)
  - Выберите новый статус из предложенных вариантов

### Для администраторов:
- **Управление сервисниками**: `/manage_services`

## 🔧 Технические особенности

### Безопасность:
- Поставщики услуг могут управлять только заявками в своих ЖК
- Проверка прав доступа на каждом этапе
- Валидация данных перед изменением статуса

### Уведомления:
- Автоматическая отправка при изменении статуса
- Персонализированные сообщения для каждого типа изменения
- Включение полной информации о заявке и контактах

### База данных:
- Отслеживание истории изменений статусов
- Связь заявок с пользователями и ЖК
- Оптимизированные запросы для производительности

## 🚀 Использование системы

### Пример рабочего процесса:

1. **Житель создает заявку** → Поставщик услуг получает уведомление
2. **Поставщик принимает в работу** (`/manage_offer_status 123` → IN_PROGRESS) → Житель получает уведомление
3. **Поставщик выполняет работу** (`/manage_offer_status 123` → COMPLETED) → Житель получает уведомление
4. **Заявка архивируется** автоматически или вручную

### Команды для тестирования:

1. Создайте заявку: `/add_offer`
2. Как поставщик услуг: `/manage_offer_status`
3. Введите номер заявки и выберите новый статус
4. Проверьте уведомления у заявителя

## 📊 Мониторинг

Система логирует все операции:
- ✅ Успешные отправки уведомлений
- ⚠️ Предупреждения (не найден поставщик, отключены уведомления)
- ❌ Ошибки (проблемы с отправкой, неверные данные)
