# 🔐 Система ролей QYZMETA Bot

## Обзор системы ролей

Платформа использует двухуровневую систему ролей:
1. **Глобальные роли** - общие роли пользователя на платформе (поле `User.role`)
2. **Локальные роли** - админы конкретных ЖК (таблица `user_jk` с полем `admins_list`)

## Глобальные роли

### Доступные роли (`UserRole`)

| Роль | Значение | Русское название | Описание |
|------|----------|------------------|----------|
| `CREATOR` | CREATOR | Создатель | Создатель бота, полные права |
| `ADMIN` | ADMIN | Администратор | Администратор платформы |
| `SUPERADMIN` | SUPERADMIN | Суперадминистратор | Расширенные права администратора |
| `MODERATOR` | MODERATOR | Модератор | Модерация контента |
| `MANAGER` | MANAGER | Менеджер | Управление заказами |
| `SUPPORT` | SUPPORT | Поддержка | Техническая поддержка |
| `PARTNER` | PARTNER | Партнер | Управление ЖК, поставщики услуг |
| `SERVICE_PROVIDER` | SERVICE_PROVIDER | Поставщик услуг | Предоставление услуг в ЖК |
| `USER` | USER | Резидент | Обычный пользователь ЖК |
| `GUEST` | GUEST | Гость | Гость без права голоса |
| `OWNER` | OWNER | Владелец | Владелец недвижимости |

## Система подачи заявок на роли

### Процесс подачи заявки

1. **FSM-форма** (`handlers/fsm/role_application_fsm.py`)
   - Ввод ФИО
   - Ввод названия компании
   - Описание цели получения роли
   - Ввод контактного телефона
   - Подтверждение заявки

2. **Сохранение в БД** (`partner_applications`)
   - Статус: `PENDING` (ожидает рассмотрения)
   - Автоматическая запись времени создания

3. **Модерация создателем** (`handlers/creator_moderation.py`)
   - Просмотр всех заявок
   - Одобрение/отклонение
   - Автоматическое назначение роли при одобрении

### Статусы заявок (`ApplicationStatus`)

| Статус | Отображение | Описание |
|--------|-------------|----------|
| `PENDING` | ⏳ Ожидает рассмотрения | Заявка подана, ждет решения |
| `APPROVED` | ✅ Одобрена | Заявка одобрена, роль назначена |
| `REJECTED` | ❌ Отклонена | Заявка отклонена |

## Команды для управления ролями

### Пользовательские команды
- `/request_role` - Подача заявки на роль
- Кнопка "🎯 Подать заявку на роль" в личном кабинете

### Команды создателя
- `/creator_panel` - Панель управления создателя
- Кнопка "📋 Заявки на партнерство" - просмотр заявок

## Технические детали

### Модели данных
- `User` - основная модель пользователя с полем `role`
- `PartnerApplication` - заявки на роли с полями:
  - `user_id` - ID пользователя
  - `requested_role` - запрашиваемая роль
  - `full_name` - ФИО заявителя
  - `company` - название компании
  - `purpose` - цель получения роли
  - `phone` - контактный телефон
  - `status` - статус заявки
  - `created_at` - время создания
  - `updated_at` - время обновления

### FSM состояния
```python
class RoleApplicationStates(StatesGroup):
    waiting_full_name = State()
    waiting_company = State()
    waiting_purpose = State()
    waiting_phone = State()
    waiting_confirmation = State()
```

### Фильтры и права доступа
- Проверка через `is_creator_by_environment(user_id)`
- Проверка ролей: `user.role in [UserRole.ADMIN, UserRole.CREATOR]`
- FSM-фильтры для предотвращения конфликтов команд
