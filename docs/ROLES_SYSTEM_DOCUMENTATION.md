# 🔐 Система ролей QYZMETA Bot

## Обзор системы ролей

Платформа использует двухуровневую систему ролей:
1. **Глобальные роли** - общие роли пользователя на платформе (поле `User.role`)
2. **Локальные роли** - админы конкретных ЖК (таблица `user_jk` с полем `admins_list`)

## Глобальные роли

### Доступные роли (`UserRole`)

| Роль | Значение | Русское название | Описание |
|------|----------|------------------|----------|
| `CREATOR` | CREATOR | Создатель | Создатель бота, полные права |
| `ADMIN` | ADMIN | Администратор | Администратор платформы |
| `SUPERADMIN` | SUPERADMIN | Суперадминистратор | Расширенные права администратора |
| `MODERATOR` | MODERATOR | Модератор | Модерация контента |
| `MANAGER` | MANAGER | Менеджер | Управление заказами |
| `SUPPORT` | SUPPORT | Поддержка | Техническая поддержка |
| `PARTNER` | PARTNER | Партнер | Управление ЖК, поставщики услуг |
| `SERVICE_PROVIDER` | SERVICE_PROVIDER | Поставщик услуг | Предоставление услуг в ЖК |
| `USER` | USER | Резидент | Обычный пользователь ЖК |
| `GUEST` | GUEST | Гость | Гость без права голоса |
| `OWNER` | OWNER | Владелец | Владелец недвижимости |

## Система подачи заявок на роли

### Процесс подачи заявки

1. **FSM-форма** (`handlers/fsm/role_application_fsm.py`)
   - Ввод ФИО
   - Ввод названия компании
   - Описание цели получения роли
   - Ввод контактного телефона
   - Подтверждение заявки

2. **Сохранение в БД** (`partner_applications`)
   - Статус: `PENDING` (ожидает рассмотрения)
   - Автоматическая запись времени создания

3. **Модерация создателем** (`handlers/creator_moderation.py`)
   - Просмотр всех заявок
   - Одобрение/отклонение
   - Автоматическое назначение роли при одобрении

### Статусы заявок (`ApplicationStatus`)

| Статус | Отображение | Описание |
|--------|-------------|----------|
| `PENDING` | ⏳ Ожидает рассмотрения | Заявка подана, ждет решения |
| `APPROVED` | ✅ Одобрена | Заявка одобрена, роль назначена |
| `REJECTED` | ❌ Отклонена | Заявка отклонена |
| `EDIT_REQUEST` | ✏️ Запрос на редактирование | Партнер запросил изменение данных |
| `DELETE_REQUEST` | 🗑️ Запрос на удаление | Партнер запросил удаление заявки |

## Система управления партнерами

### Команда `/is_partner`
Проверяет статус партнера и предоставляет доступ к партнерской панели:

**Для неавторизованных пользователей:**
- Информация о роли партнера
- Inline-кнопка "📝 Стать партнером" для подачи заявки

**Для одобренных партнеров:**
- Персонализированное приветствие
- Reply-клавиатура `PARTNER_PANEL_KB` с функциями

### Партнерская панель (`PARTNER_PANEL_KB`)

| Кнопка | Функция | Модуль |
|--------|---------|--------|
| ЖК под управлением | Управление жилыми комплексами | `partner_panel.py` |
| Аналитика | Статистика и отчеты | `partner_panel.py` |
| Кабинет партнера | Управление профилем и заявкой | `partner_panel.py` |
| Поддержка | Контакты службы поддержки | `partner_panel.py` |
| Выйти из режима партнера | Возврат к основному меню | `partner_panel.py` |

### Контроль доступа

**Декоратор `@partner_access_required`:**
```python
def partner_access_required(func):
    """Проверяет наличие одобренной заявки партнера"""
    async def wrapper(message: Message, state: FSMContext, session: AsyncSession, **kwargs):
        if not await is_approved_partner(session, message.from_user.id):
            await message.answer("❌ У вас нет доступа к партнерской панели.")
            return
        return await func(message, state, session, **kwargs)
    return wrapper
```

**Функция проверки статуса:**
```python
async def is_approved_partner(session: AsyncSession, user_id: int) -> bool:
    """Проверяет, является ли пользователь одобренным партнером"""
    stmt = select(PartnerApplication).where(
        PartnerApplication.user_id == user_id,
        PartnerApplication.status == ApplicationStatus.APPROVED
    ).limit(1)
    result = await session.execute(stmt)
    return result.scalar_one_or_none() is not None
```

## Команды для управления ролями

### Пользовательские команды
- `/is_partner` - Проверка статуса партнера и доступ к панели
- Inline-кнопка "📝 Стать партнером" - подача заявки на партнерство
- Reply-клавиатура партнера для навигации по функциям

### Команды создателя
- `/creator_panel` - Панель управления создателя
- Кнопка "📋 Заявки на партнерство" - просмотр и модерация заявок

### Партнерские команды (через reply-клавиатуру)
- "ЖК под управлением" - управление жилыми комплексами
- "Аналитика" - статистика и отчеты
- "Кабинет партнера" - управление профилем
- "Поддержка" - контакты службы поддержки
- "Выйти из режима партнера" - возврат к основному меню

## Технические детали

### Модели данных
- `User` - основная модель пользователя с полем `role`
- `PartnerApplication` - заявки на роли с полями:
  - `user_id` - ID пользователя Telegram
  - `requested_role` - запрашиваемая роль (enum `UserRole`)
  - `full_name` - ФИО заявителя
  - `company` - название компании/организации
  - `purpose` - цель получения роли и планы использования
  - `phone` - контактный телефон
  - `status` - статус заявки (enum `ApplicationStatus`)
  - `created_at` - время создания заявки
  - `updated_at` - время последнего обновления

### Клавиатуры
- `PARTNER_PANEL_KB` - reply-клавиатура с 5 кнопками партнерских функций
- `MAIN_KB` - основная клавиатура для обычных пользователей
- Inline-клавиатуры для подачи заявок и подтверждений

### Роутеры и обработчики
- `handlers/platform_roles/partner_role_handler.py` - команда `/is_partner`
- `handlers/partner_panel.py` - обработчики партнерской панели
- `handlers/fsm/role_application_fsm.py` - FSM подачи заявок
- `handlers/creator_moderation.py` - модерация заявок создателем

### FSM состояния

class RoleApplicationStates(StatesGroup):

### Фильтры и права доступа
- Проверка через `is_creator_by_environment(user_id)`
- Проверка ролей: `user.role in [UserRole.ADMIN, UserRole.CREATOR]`
- Декоратор `@partner_access_required` для партнерских функций
- Проверка одобренных заявок через `is_approved_partner(session, user_id)`
- FSM-фильтры для предотвращения конфликтов команд
- Сигнатуры обработчиков с `**kwargs` для совместимости с middleware
