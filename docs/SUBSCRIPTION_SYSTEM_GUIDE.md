# Система подписок и монетизации Qyzmeta Bot

## 📋 Обзор

Система подписок реализует монетизацию платформы через ограничение количества адресов для регистрации пользователей. Это первый этап внедрения бизнес-модели в Qyzmeta Bot.

## 🎯 Основные возможности

### Тарифные планы

| Тариф | Адреса | Цена/мес | Особенности |
|-------|--------|----------|-------------|
| 🆓 FREE | 1 | Бесплатно | Базовый функционал |
| 🥉 BASIC | 3 | 2,990 ₸ | Приоритетные уведомления |
| 🥈 PREMIUM | 10 | 5,990 ₸ | Эксклюзивные предложения, аналитика |
| 👑 VIP | Неограниченно | 9,990 ₸ | VIP статус, персональный консультант |

### Ограничения

- **Жители**: могут зарегистрироваться только по одному адресу бесплатно
- **Дополнительные адреса**: только через платную подписку
- **Автоматическая проверка**: при попытке регистрации нового адреса

## 🔧 Техническая архитектура

### База данных

#### Таблица `user_subscriptions`
```sql
- id (PK)
- user_id (FK → users.user_id) 
- tier (ENUM: FREE, BASIC, PREMIUM, VIP)
- status (ENUM: ACTIVE, EXPIRED, CANCELLED, PENDING)
- max_addresses (INT)
- started_at (TIMESTAMP)
- expires_at (TIMESTAMP, nullable)
- payment_info (TEXT, nullable)
- notes (TEXT, nullable)
- created_at/updated_at (TIMESTAMP)
```

#### Индексы
- `ix_user_subscriptions_user_id`
- `ix_user_subscriptions_status` 
- `ix_user_subscriptions_tier`
- `ix_user_subscriptions_expires_at`
- `ix_user_subscriptions_user_status_active` (составной)

### Enums

#### SubscriptionTier
```python
FREE = "FREE"      # 1 адрес, 0 ₸
BASIC = "BASIC"    # 3 адреса, 2,990 ₸  
PREMIUM = "PREMIUM" # 10 адресов, 5,990 ₸

```

#### SubscriptionStatus
```python
ACTIVE = "ACTIVE"       # Активная подписка
EXPIRED = "EXPIRED"     # Истекшая подписка
CANCELLED = "CANCELLED" # Отмененная подписка

```

## 🚀 Основные компоненты

### 1. Модели данных
- `database/models/model_user_subscription.py` - ORM модель подписки
- `database/enums/subscription_enums.py` - Перечисления тарифов и статусов
- `database/models/orm_user_subscription.py` - ORM функции для работы с подписками

### 2. Бизнес-логика
- `services/subscription_service.py` - Сервис управления подписками
- Проверка лимитов адресов
- Статистика и аналитика
- Управление тарифами

### 3. Пользовательский интерфейс
- `keyboards/subscription_keyboards.py` - Клавиатуры для подписок
- `handlers/subscription_handlers.py` - Обработчики callback'ов
- `handlers/user_private.py` - Команда "Моя подписка" (доступна через "Мой профиль")
- Интеграция с существующей кнопкой "Мой дом 🏢" для просмотра адресов

### 4. Административная панель
- `handlers/admin_private.py` - Админ панель бизнес-моделей
- Управление подписками пользователей
- Статистика доходов
- Мониторинг истекающих подписок

## 🔄 Процессы

### Апгрейд подписки
1. Пользователь достигает лимита адресов
2. Система предлагает апгрейд
3. Выбор тарифа и срока
4. Подтверждение платежа (пока тестовый режим)
5. Автоматическое обновление лимитов

### Управление просроченными подписками
1. Автоматическая проверка истекших подписок
2. Перевод статуса в EXPIRED
3. Ограничение функционала до уровня FREE
4. Уведомления о необходимости продления

## 🛠️ Troubleshooting

### Распространенные ошибки

#### KeyError: 'payment_info'
**Проблема**: Ошибка при обращении к информации о платеже в subscription_info
**Решение**: Использовать `.get("payment_info")` вместо прямого обращения к ключу

#### TypeError: can't compare offset-naive and offset-aware datetimes
**Проблема**: Несоответствие типов datetime при сравнении времени истечения подписки
**Решение**: Использовать `datetime.now(timezone.utc)` вместо `datetime.utcnow()`

#### Таблица user_subscriptions не найдена
**Проблема**: База данных не содержит таблицу подписок
**Решение**: Запустить `python scripts/create_tables.py`

#### Пользователь не может зарегистрировать адрес
**Проблема**: Лимит адресов исчерпан, но система не предлагает апгрейд
**Решение**: Проверить импорт subscription_service в обработчиках

#### Как посмотреть свои адреса?
**Решение**: Используйте кнопку "Мой дом 🏢" в главном меню - там отображаются все зарегистрированные адреса

#### Где найти "Моя подписка"?
**Решение**: Кнопка "Моя подписка 💎" находится в меню "Мой профиль 👤" → "Моя подписка 💎"
