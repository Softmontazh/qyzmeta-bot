# 🌐 LotBox Platform - Манифест экосистемы

## 🎯 Концепция платформы

**LotBox** - универсальная цифровая платформа нового поколения для ведения строительной инфраструктуры с революционным подходом "снизу вверх".

### 🔥 Ключевая идея
Создание **цифрового паспорта объекта** по ходу работы, а не после завершения. Живой CRM в поле, а не в офисе.

## 🏗️ Архитектура экосистемы

### 📦 Проекты платформы

#### 1. **LotBox-Bot** (Основная платформа)
- **Репозиторий:** `lotbox-bot`
- **Назначение:** Универсальная система лотов для строительной отрасли
- **Пользователи:** Строители, монтажники, подрядчики
- **Функции:**
  - 🔨 Создание лотов-задач (задачи, проекты, дефекты)
  - 🛒 Маркетплейс (продажа/покупка/обмен/аренда материалов)
  - 🔗 Привязки лотов (создание цифровой истории WEB3.0)
  - 📋 Управление проектами в Telegram группах
  - 📍 Привязка к физическим объектам

#### 2. **Qyzmeta-Bot** (ЖКХ интерфейс)
- **Репозиторий:** `qyzmeta-bot` (текущий проект)
- **Назначение:** Облегченный интерфейс для жителей и управления ЖКХ
- **Пользователи:** Жители, УК, ОСИ/КСК, сервисные компании
- **Функции:**
  - 🏠 Заявки от жителей (как лоты)
  - 🏢 Управление ЖК
  - 🔍 Доступ к цифровым паспортам объектов
  - 📱 Простой интерфейс для жителей

#### 3. **MicrofirmKZ** (CRM + Автоматизация)
- **Репозиторий:** `microfirm-kz`
- **Назначение:** CRM для начинающих предпринимателей + автоматизация документооборота
- **Пользователи:** УК, сервисные компании, подрядчики, ИП
- **Функции:**
  - 🏢 Регистрация и верификация компаний
  - 🤝 Автоматические привязки (УК↔Сервисы↔ЖК)
  - 📄 Генерация документов (договоры, акты, дефектные ведомости)
  - 💼 CRM функционал для малого бизнеса

## 🎯 Центральная модель: LOT

**Лот** - уникальная абстрактная единица, которая может представлять:
- 📋 Задачу/проект
- 🛍️ Товар/услугу
- 🔄 Обмен/аренду
- 💰 Сделку
- 🏠 Заявку ЖКХ
- 🔧 Дефект/проблему
- 📦 Что угодно другое

### 🔗 UUID и WEB3.0
Каждый лот имеет UUID и может быть привязан к другим лотам, создавая цифровую историю объекта - основу для WEB3.0 технологий.

## 👥 Пользователи платформы

### Единая модель User для всех проектов:
- **Базовая информация:** user_id, имя, username, язык
- **Telegram атрибуты:** is_bot, is_premium, is_business
- **Контакты:** phone, email, city
- **Статусы:** is_banned, is_blocked, подписки на каналы/группы
- **Реферальная система:** invited_by_id
- **Активность:** last_active, wants_notifications

### Роли в экосистеме:
- **CREATOR** - Создатель платформы (безлимитные права)
- **OWNER** - Владелец бизнеса/объекта
- **ADMIN/SUPERADMIN** - Администраторы разных уровней
- **MANAGER** - Менеджеры компаний
- **USER** - Обычные пользователи (жители, работники)
- **PARTNER** - Партнеры платформы
- **GUEST** - Гости (ограниченный доступ)

### Лимиты по ролям:
Система `lot_limits` контролирует количество лотов для каждой роли, обеспечивая справедливое использование ресурсов.

## 🔄 Workflow взаимодействия

1. **LotBox-Bot:** Строитель создает лот-задачу → привязывает к проекту/зданию
2. **MicrofirmKZ:** Компании регистрируются → настраивают привязки к ЖК
3. **Qyzmeta-Bot:** Житель создает заявку → система автоматически уведомляет нужную компанию
4. **Цифровой след:** Все лоты связываются через UUID → создается полная история объекта

## 🛠️ Техническая архитектура

### Shared модуль (общий для всех проектов):
```
shared/
├── database/
│   ├── models/
│   │   ├── lot.py           # Центральная модель лотов
│   │   ├── user.py          # Обширная модель пользователей
│   │   ├── company.py       # Компании (УК, сервисы, подрядчики)
│   │   ├── jk.py           # Жилые комплексы
│   │   ├── user_jk.py      # Привязки пользователей к ЖК
│   │   ├── offer.py        # Заявки/предложения (наследуют от лотов)
│   │   ├── lot_limit.py    # Лимиты лотов по ролям
│   │   └── binding.py      # Различные привязки
│   ├── enums/
│   │   ├── user_enums.py   # Роли и языки пользователей
│   │   ├── lot_enums.py    # Типы и статусы лотов
│   │   └── offer_enums.py  # Статусы заявок
│   └── services/
│       ├── lot_service.py          # Работа с лотами
│       ├── user_service.py         # Работа с пользователями
│       ├── web3_service.py         # UUID цепочки и WEB3.0
│       ├── binding_service.py      # Автоматические привязки
│       ├── notification_service.py # Уведомления между проектами
│       └── document_service.py     # Генерация документов
└── utils/
    ├── web3_utils.py      # WEB3.0 утилиты и UUID
    ├── digital_passport.py # Цифровые паспорта объектов
    ├── role_utils.py      # Утилиты для работы с ролями
    └── formatters.py      # Форматирование данных
```

### Виртуальные окружения:
- **Каждый проект** имеет свою venv
- **Shared модуль** подключается как `-e ../shared/`
- **База данных** единая для всей платформы
