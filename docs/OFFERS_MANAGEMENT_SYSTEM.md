# Система управления заявками для поставщиков услуг

## Обзор

Полностью переработанная система управления заявками с меню-ориентированным интерфейсом вместо ввода номеров заявок.

## Основные изменения

### 🔄 Изменения в интерфейсе
- **Кнопка**: `"Управление статусами заявок 📝"` → `"Управление заявками 📋"`
- **Принцип работы**: Ввод номера заявки → Меню с категориями статусов

### 📋 Новое меню управления заявками

При нажатии кнопки "Управление заявками 📋" открывается меню:

1. **🆕 Активные заявки** - новые поступившие заявки
2. **⏳ Заявки в работе** - принятые в работу заявки  
3. **❌ Отмененные** - отмененные заявки
4. **✅ Выполненные** - завершенные заявки

### 🎯 Функциональность по статусам

#### Активные заявки
- **Действия**: Принять в работу | Отклонить
- **Переходы**: ACTIVE → IN_PROGRESS | CANCELLED

#### Заявки в работе  
- **Действия**: Завершить | Отменить
- **Переходы**: IN_PROGRESS → COMPLETED | CANCELLED

#### Отмененные заявки
- **Действия**: Возобновить
- **Переходы**: CANCELLED → IN_PROGRESS

#### Выполненные заявки
- **Действия**: Только просмотр (архивные)

### 📄 Пагинация
- **Лимит**: 2 заявки на страницу
- **Навигация**: ◀️ Назад / Вперед ▶️
- **Автоматическая**: При превышении лимита

### 🔍 Детальный просмотр заявки

Для каждой заявки доступна полная информация:
- ЖК и квартира
- Данные заявителя (имя, телефон, username)
- Название и описание заявки
- Дата создания и текущий статус
- Медиафайлы (фото/видео)
- Кнопки действий в зависимости от статуса

### 🎛️ Контролы и взаимодействие

#### Основные кнопки
- **📞 Связаться** - прямая ссылка на пользователя
- **📎 Посмотреть медиа** - просмотр прикрепленных файлов
- **🔙 К списку** - возврат к списку заявок
- **🔙 Назад к меню** - возврат к выбору категории

#### Медиафайлы
- Интеграция с BUS системой (`bus_media_id`)
- Поддержка фото и видео
- Отправка в личные сообщения сервисника

## Техническая реализация

### 📁 Измененные файлы

#### `keyboards/reply.py`
```python
MANAGE_OFFER_STATUS_KB = get_keyboard(
    "Управление заявками 📋",  # Новый текст кнопки
    "Мои сервисы в ЖК 📝",
    "Моя статистика 📊", 
    "Выход из режима сервисника",
    placeholder="Управление заявками",
    sizes=(1, 1, 1, 1, 1),
)
```

#### `handlers/service_provider_panel.py`
- **Новые обработчики**:
  - `show_offers_by_status()` - показ заявок по статусу
  - `show_offers_list()` - список с пагинацией
  - `handle_offers_pagination()` - навигация по страницам
  - `back_to_offers_menu()` - возврат к меню
  - `manage_single_offer()` - детальный просмотр
  - `change_offer_status()` - изменение статуса
  - `view_offer_media()` - просмотр медиа

#### `database/models/orm_offer.py` 
- **Новая функция**: `orm_get_offers_by_status_for_provider()`
  - Фильтрация по поставщику услуг
  - Пагинация (page, limit)
  - Загрузка связанных данных (User, JK, UserJK)
  - Универсальная обработка статусов

### 🔄 Callback Data Structure

```
offers_status:{status_key}           # Выбор категории
offers_page:{status_key}:{page}      # Пагинация  
offer_details:{offer_id}:{status_key} # Детали заявки
change_status:{offer_id}:{new_status}:{status_key} # Изменение статуса
view_media:{offer_id}                # Просмотр медиа
back_to_offers_menu                  # Возврат к меню
```

### 🎯 Enum обработка

Универсальная система конвертации статусов:
- `"active"` ↔ `OfferStatus.ACTIVE`
- `"in_progress"` ↔ `OfferStatus.IN_PROGRESS`  
- `"completed"` ↔ `OfferStatus.COMPLETED`
- `"cancelled"` ↔ `OfferStatus.CANCELLED`

## Интеграции

### 🔗 BUS Media System
- Обмен медиафайлами между ботами
- Автоматический fallback при ошибках
- Поддержка фото и видео форматов

### 📬 Notification Service  
- Уведомления пользователей об изменении статуса
- Интеграция с системой изменения статусов
- Контекстные сообщения в зависимости от статуса

### 🏢 Service Provider Access Control
- Фильтрация заявок по ЖК поставщика
- Проверка прав доступа на каждом уровне
- Отображение только релевантных заявок
