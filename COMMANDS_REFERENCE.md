## 🔒 **Система доступа**

### **Проверка регистрации:**
Большинство функциональных команд требуют завершенной регистрации:
- Пользователь должен иметь роль отличную от `GUEST`
- Пользователь должен иметь номер телефона в БД

### **Административный доступ:**
- Проверяется через фильтр `IsAdmin()` в `filters/chat_types.py`
- Использует переменную `PLATFORM_ADMIN_IDS` из `.env`

### **Фильтры чатов:**
- `ChatTypeFilter(["private"])` - только личные сообщения
- `ChatTypeFilter(["group", "supergroup"])` - только групповые чаты

## 🗂️ **Структура обработчиков**

### **Основные роутеры:**
1. **`user_private_router`** - личные сообщения от пользователей
2. **`admin_router`** - личные сообщения от администраторов  
3. **`user_group_router`** - групповые чаты
4. **`manage_jk_router`** - управление ЖК

### **FSM роутеры:**
1. **`add_offer_router`** - создание заявок
2. **`my_offers_router`** - управление заявками
3. **`add_jk_router`** - добавление ЖК
4. **`add_lot_router`** - создание лотов
5. **`search_lot_router`** - поиск лотов
6. **`user_to_jk_router`** - привязка к ЖК

## 📚 **Дополнительная информация**

### **Альтернативы командам:**
Большинство команд дублируются кнопками в клавиатуре:
- "🏠 Главная" = `/main_menu`
- "Мой профиль 👤" = `/my_profile`
- "Мой дом 🏢" = `/my_jk`
- "Создать заявку 📝" = `/add_offer`

### **Состояния FSM:**
Многие команды переводят бота в FSM состояния для пошагового сбора данных:
- Создание заявок (AddOffer states)
- Добавление ЖК (JKCreationState)
- Добавление лотов (AddLot states)
- Поиск лотов (SearchLot states)

### **Безопасность:**
- Все команды с доступом к функционалу проверяют регистрацию через `check_user_registration()`
- Административные команды защищены фильтром `IsAdmin()`
- FSM состояния можно сбросить командой `/отмена`
